# Rudie Orchestrator Platform

## Installation

```bash
git clone https://github.com/agilexplus-rv/starter-templates.git my-project
cd my-project
rm -rf .git && git init
git add . && git commit -m "Initial commit from Rudie Orchestrator template"
```

Then connect your own repo and push:

```bash
git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO.git
git branch -M main && git push -u origin main
```

See `README-baby-steps.md` for the full setup walkthrough.

## Principle

You provide **INTENT**
The Orchestrator generates **STRUCTURE**
Execution agents write **CODE**

**Claude orchestrates. Antigravity implements. Gemini ideates UI.**
Claude never writes production code. This is non-negotiable.

See `docs/ai/claude-role-constraints.md` for the full boundary definition.

---

## What Is Antigravity?

Antigravity is the AI implementation agent layer. It receives structured mission
files generated by Claude and executes them: writing components, running
migrations, wiring APIs, and implementing everything the architecture specifies.

Antigravity is the *only* agent authorised to write, modify, or delete source
code in this repository.

For setup and usage, see `docs/ai/antigravity-agent-guide.md`.

## Development Workflow

```
1. Write requirements
   → docs/ai/requirements.md (use template)

2. Start Claude orchestrator
   → "Use docs/ai/requirements.md and run the wizard."
   → Claude validates requirements, generates architecture docs and missions

3. [Optional] UI ideation
   → Add screenshots to docs/ui-references/
   → "Use docs/ui-references/homepage.png as UI reference for the homepage."
   → Gemini generates concept → Claude validates → UI missions added

4. Review missions
   → Read docs/ai/missions/ — approve each mission before execution

5. Execute missions in Antigravity
   → See docs/ai/antigravity-agent-guide.md
   → Respect docs/ai/agent-concurrency-protocol.md for parallel runs

6. Run preflight
   → ./ai/checks/preflight.sh
   → All checks must pass

7. Deploy to staging, run smoke test

8. Approve and deploy to production
```

## Screenshot to Component Workflow

1. Generate layout in Stitch or Figma
2. Save screenshot to `docs/ui-references/`
3. Ask the orchestrator:

   ```
   Use docs/ui-references/homepage.png as UI reference for the homepage.
   ```

The orchestrator will:
- Pass the screenshot to Gemini for UI concept generation
- Validate the concept against `system-architecture.md`
- Generate a UI mission for Antigravity
- Register the component in `docs/ai/ui-component-registry.md` on completion

## Preflight

Run before every deployment:

```bash
./ai/checks/preflight.sh
```

Checks (in order):
1. TypeScript type check
2. Lint (zero warnings)
3. Application build
4. Test suite
5. Prisma schema validation
6. Migration destructiveness check
7. Secrets scan
8. Environment variable completeness
9. Docker build

**Do not deploy if any check fails.**

## Compliance Projects

### Profiles

Profiles adjust the framework's architecture requirements, approval gates,
and documentation obligations to match your project type.

| Profile | Use when | File |
|---|---|---|
| **Standard** | Default — typical web app, no special constraints | `docs/ai/profiles/standard.md` |
| **Startup MVP** | Pre-revenue, speed matters, 1–3 person team | `docs/ai/profiles/startup-mvp.md` |
| **API-Only** | No frontend — backend service or microservice | `docs/ai/profiles/api-only.md` |
| **UI-Heavy** | 20+ components, design system, Gemini pipeline elevated | `docs/ai/profiles/ui-heavy.md` |
| **Regulatory-Heavy** | HIPAA, GDPR, PCI-DSS, financial or medical data | `docs/ai/profiles/regulatory-heavy.md` |

### Activating a profile

Load the profile file alongside your requirements at the start of any Claude session:

```
Use docs/ai/requirements.md and run the wizard.
Load docs/ai/claude-role-constraints.md.
Load docs/ai/profiles/startup-mvp.md — use this profile.
```

Standard is the default and requires no explicit loading.
Profiles can be combined where it makes sense (e.g. API-Only + Regulatory-Heavy
for a HIPAA-compliant backend service).
